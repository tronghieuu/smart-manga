HTML5 document type

Some HTML elements and CSS properties used by Bootstrap need to be set to HTML5 document type. Each page in your project should be set according to the following format.

Mobile first

In Bootstrap 2, we added mobile-friendly styles to some key parts of the framework. And in Bootstrap 3 In, we rewrote the entire framework to make it mobile friendly from the beginning. This time, instead of simply adding some optional styles for mobile devices, it is directly integrated into the core of the framework. In other words, Bootstrap It is mobile first. The style for mobile devices is integrated into every corner of the frame instead of adding an extra file.

Typography and links

Bootstrap typography and link style set basic global styles. They are: set background-color for the body element: #fff; use @font-family-base, @font-size-base and @line-height-base a variables are used as the basic parameters of typesetting, and the basic colors are set for all links @link-color, and only add an underline when the link is in the :hover state. These styles can be found in the corresponding source code in the scaffolding.less file.

Normalize.css

In order to enhance the consistency of cross-browser performance, we use Normalize.css, a CSS maintained by Nicolas Gallagher and Jonathan Neal Reset the style library.

Layout container

Bootstrap needs to wrap a .container container for the page content and grid system. We provide two classes for this purpose. Note that due to padding and other attributes, these two Container classes cannot be nested within each other.

Grid system

Bootstrap A responsive, mobile device-first streaming grid system is provided. As the size of the screen or viewport increases, the system will automatically be divided into up to 12 columns. It contains easy-to-use predefined classes and powerful mixins Used to generate a more semantic layout.
